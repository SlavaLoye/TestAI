Короткий README

Как пользоваться (поведение приложения)
• Запуск: приложение стартует с AppRootView (см. TEst_AiApp). Профиль по умолчанию инициализируется из сидов (UsersSeed), приоритетно выбирается пользователь «Вячеслав».
• Профиль/Настройки:
   • Экран SettingsView показывает «Мой аккаунт»: имя, дата рождения, пол, иконка по полу.
   • Тап по карточке или кнопка «Редактировать» открывает форму. Можно изменить имя, дату рождения и пол. Сохранение обновляет локальные данные экрана; primaryUserID синхронизируется в ProfileStore и зеркально в @AppStorage("primaryUserID").
   • Если главного пользователя нет, при первом появлении экран назначит «Вячеслава» или первого доступного из сидов.
• Выход:
   • В секции «Учетная запись» есть кнопка «Выйти». Реальная логика передаётся через onLogout из родителя (например, AppRootView).
• О приложении:
   • Секция показывает версию из CFBundleShortVersionString.

Архитектура
MVVM 
• UI: SwiftUI
   • SettingsView — экран профиля/настроек с редактированием через sheet. Делегирует выход через onLogout.
• Модели:
   • User — сущность пользователя (id, name, birthDate, gender).
   • Gender — enum пола с локализуемыми title и символами SF Symbols.
• Данные:
   • UsersSeed — набор демонстрационных пользователей.
   • ProfileStore — фасад над UserDefaults для полей профиля: primaryUserID, name, birthDate, gender, email и пр. Имеет ensureDefaultProfile() для bootstrap.
• Точка входа:
   • TEst_AiApp — @main, поднимает WindowGroup с AppRootView.

Что бы улучшил при большем времени
• Единый источник правды для пользователей:
   • Вынести список пользователей из локального @State в UsersStore (ObservableObject или actor) с персистентностью. Он будет:
      • Хранить массив пользователей и primaryUserID.
      • Предоставлять computed primaryUser.
      • Инкапсулировать сохранение правок.
• Персистентность правок:
   • Сейчас сохраняется только primaryUserID. Добавить запись имени/даты/пола в ProfileStore (и/или UsersStore + файл/БД).
• MVVM:
   • Вынести бизнес-логику редактирования (валидация, сохранение, преобразования) в ViewModel (ObservableObject), чтобы упростить SettingsView.
• Локализация:
   • Вынести все строки в Localizable.strings и покрыть минимум ru/en.
   • Использовать Date.FormatStyle для локализованного отображения дат.
• Валидация:
   • Имя — тримминг, длина, допустимые символы.
   • Дата рождения — не в будущем, разумные границы.
• Тесты:
   • Модульные тесты для ProfileStore.ensureDefaultProfile и геттеров/сеттеров.
   • Тесты на UsersStore (если добавлен).
   • UI-тест сценария редактирования.
   • Использовать Swift Testing (@Suite/@Test).
• Доступность:
   • accessibilityLabel/Hint для кнопок, поддержка Dynamic Type.
• UX/Визуал:
   • Разные символы/иконки для male/female, возможность аватара-фото.
   • Улучшить карточку профиля (карточный фон, состояние загрузки/сохранения).
• Инициализация:
   • Явно вызывать ProfileStore.ensureDefaultProfile() при старте (в AppRootView.onAppear или App.init), чтобы гарантировать корректные дефолты.
• Безопасность:
   • Перенос чувствительных данных (если появятся) в Keychain.
